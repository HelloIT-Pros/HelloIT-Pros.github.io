CRM Technical Notes

Entry Trigger: New order placed by customer

Last Order Date System:
• Record date of most recent order
• All journey timing based on days AFTER last order date
• Simple, predictable timing structure
• Updates automatically with each new order

Loop Mechanism:
• At ANY decision point, if order is placed → Journey restarts from Stage 0
• Last order date updated to new order date
• Creates self-perpetuating engagement cycle
• Prevents customers from falling off
• Journey extends up to 210 days from last order before transitioning to Win-Back

Personalization Requirements:
• All rep emails use conversational, personal tone
• Product recommendations based on purchase history
• Practice name used throughout messaging
• Rep includes personal notes when possible
• Hand-picked selections show rep's attention and care
• Mix of product showcases with relationship-building check-ins

Status Updates:
• Entry: Active Customer
• Throughout journey (Stages 0-15): Active Customer (as long as engaging)
• After Stage 15 (no order): Inactive Customer → Exit to Win-Back Journey

Journey Philosophy:
• Build genuine relationships, not just transactional contacts
• Alternate between product showcases and personal check-ins
• Show customers they're valued beyond their purchases
• Rep becomes a trusted partner, not just a salesperson
• 16 touchpoints over ~7 months to maximize retention

Shopify Integration Points

1. Real-Time Order Monitor (All Stages):
• Shopify webhook: order.created
• Match to customer email OR phone
• If customer in Active Journey → Immediately restart journey from beginning
• Update Last Order Date to new order date
• Send rep notification: "[Customer] placed a new order!"

2. Last Order Date Tracker:
• CRM field: "Last Order Date" (date field, auto-populated)
• Automatically updates with each new order
• All stage timing calculated from this date
• Example: Stage 3 triggers on Last Order Date + 21 days

3. Purchase History Analysis:
• Pull past order data for personalized product recommendations
• Identify favorite styles, price points, quantities
• Rep reviews before sending personalized stages

4. Rep Notification System:
• When customer places new order, send notification to rep
• Email subject: "[Customer Name] just placed a new order!"
• Include order details and journey restart confirmation

Data Matching Priority:
1. Customer ID (primary)
2. Email address (secondary)
3. Phone number (tertiary)
Always log match method for audit trail

Active Integration Summary:
• Real-Time Order Monitor: Shopify webhooks monitoring for journey restart
• Last Order Date Tracker: Simple date-based timing system
• Purchase History Engine: Personalized product recommendations
• Rep Notification System: Keep reps informed of customer activity
